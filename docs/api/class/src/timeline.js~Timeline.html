    public add(event: Object): Timeline source

    Adds an event to the timeline.

    An event is any object that implements a basic interface consisting of start and stop methods. start is called with a completion callback function that the event must call when completion is reached. stop should stop any asynchronous work for the event and should not invoke the complete method passed into start.

    public branch(fn: Function): Timeline source

    Creates a number of timelines, passes them to the function provided, and wraps them all in a ConcurrentEvent.

    The function passed in must have a defined .length. That length is used to know how many new timelines to create. This mainly means your function should not use rest parameters.

    Params:

    NameTypeAttributeDescription
    fn Function

    A function called that is expected to configure the newly created timelines.

    Return:

    Timeline

    The timeline, to support call chaining.

    See:

    • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/length

    public start(options: Object) source

    Starts playing the timeline.

    Params:

    NameTypeAttributeDescription
    options Object

    Playback options.
    options.iterations Number

    The number of times to play through the events. Defaults to 1.
    options.onComplete Function

    A function to invoke when the timeline finishes playing. This will never be invoked if iterations is set to Infinity.

    public stop() source

    Stops the timeline, if it is playing.

    Aside from preventing starting any new events, this also calls stop on the current event to allow it to clean up.
